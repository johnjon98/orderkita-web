# Phase 05 — UI Primitives

## Goal
Install and configure all shadcn/ui base components. These are pure, logic-free building blocks with no knowledge of the OrderKita domain. Every component in `common/`, `customer/`, `merchant/`, and `kitchen/` builds on top of these.

## Location
`src/components/ui/` — auto-generated by shadcn/ui CLI, do not hand-edit.

## Components to Install

### Layout & Shell
| Component | Usage |
|---|---|
| `Card` | Order cards, menu item cards, KDS tickets |
| `Sheet` | Cart drawer (customer), order detail (merchant) |
| `Dialog` | Confirmation modals, item detail popup |
| `Drawer` | Mobile cart, mobile order detail |
| `ScrollArea` | KDS board scroll, long order lists |
| `Separator` | Visual dividers in cards and panels |
| `Tabs` | Dashboard sections (orders / menu / settings) |

### Forms & Inputs
| Component | Usage |
|---|---|
| `Button` | All CTAs — variants: default, outline, ghost, destructive |
| `Input` | Text inputs everywhere |
| `Label` | Form labels |
| `Select` | Status filter, category select, state dropdown |
| `Form` | React Hook Form integration wrapper |
| `Checkbox` | Modifier selection (multi-select) |
| `RadioGroup` | Modifier selection (single-select) |
| `Switch` | SST toggle, item availability, store active toggle |
| `Textarea` | Order notes |

### Data Display
| Component | Usage |
|---|---|
| `Badge` | Status chips, modifier price labels |
| `Avatar` | User/merchant avatar |
| `Skeleton` | Loading states for all cards and lists |
| `Progress` | Order preparation progress indicator |
| `Alert` | Error banners, info notices |
| `Tooltip` | Icon button labels |

### Feedback & Navigation
| Component | Usage |
|---|---|
| `Toast` / `Sonner` | Success/error notifications |
| `Command` | Search within menu editor, order search |
| `Popover` | Date pickers, filter dropdowns |
| `DropdownMenu` | User menu, action menus on rows |

## Custom Theme

Extend the default shadcn/ui theme in `tailwind.config.ts`:

```ts
colors: {
  // Brand
  brand: {
    DEFAULT: '#E85D04',    // warm orange — primary CTA
    foreground: '#ffffff',
  },
  // Status colors (used in StatusBadge)
  status: {
    pending:   '#F59E0B',  // amber
    accepted:  '#3B82F6',  // blue
    preparing: '#8B5CF6',  // purple
    ready:     '#10B981',  // green
    completed: '#6B7280',  // grey
    cancelled: '#EF4444',  // red
  }
}
```
